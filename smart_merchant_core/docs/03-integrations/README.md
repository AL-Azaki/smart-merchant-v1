# 03-integrations
## وصف المجلد
يحتوي على وثائق التكامل (Integration) بين النطاقات المختلفة.
## الغرض منه
تنظيم وضبط التفاعلات بين Domains دون حدوث تداخل أو كسر لمبادئ Clean Architecture.
## قائمة الوثائق
(وثائق الـ Integrations مثل Sales-Finance, Purchasing-Inventory).
## ترتيب قراءة الوثائق
تُقرأ حسب الحاجة لكل تكامل بين نطاقين.
## Dependencies
يعتمد بشكل كلي على الدساتير (Foundations) والكيانات (Entities) المعرّفة في 02-domains.
